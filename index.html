[daVinci, Figure 3] 所有预训练数据源在Data Darwinism L0-L9分类体系上的映射

2.4 关键消融实验发现

(1) 数据处理深度
[daVinci, Table 7] L2 vs L3 代码过滤对比: MBPP +3.40, Overall +0.27
[daVinci, Table 8] L4数学精炼对比: MATH +7.00, GSM-8K +1.37, Overall +0.56。使用Qwen3-235B进行生成式精炼。
[daVinci, Figure 7] CC-QA和CodeQA合成数据的域特异性效果: CodeQA提升编程, CC-QA提升科学。
(2) 训练动态: 饱和速率差异
[daVinci, Figure 8] Stage 1的6T训练轨迹。General knowledge在~1T tokens内快速饱和; Code和Science在4T+仍持续增长。
[daVinci, Figure 9] Stage 1-3(继续调比例) vs Stage 2(引入QA)的对比: Science +20.34 vs +15.75, Code +2.62 vs +2.15。"引入QA显著优于仅调整比例"。
(3) 数据混合设计
[daVinci, Figure 11a] Code-Science比例分析(雷达图): C-30-S-30(均衡)优于C-40-S-20(偏向代码)。QA固定30%，其余为CC。
[daVinci, Figure 11b] Stage 2-1中QA 10%-100%的效果: General在100%时急剧下降; Code在30%后非单调。
[daVinci, Table 9] Stage 2-2中QA 30%/50%/70%均单调提升。70% QA: Overall 49.84。
结论: 渐进式混合——先保守(30% QA)建基础，再激进(70% QA)增强。[daVinci, Section 4.3.2]
(4) 评估方法的影响
[daVinci, Figure 12] MMLU上PPL评估 vs Generative评估导致不同模型排名。含QA训练的模型在Generative评估中表现不成比例地好。

[OLMo 3, Figure 2] 模型流程划分为: 预训练(左) → 后训练(右)，各含多个子阶段。

[OLMo 3, Table 4] Dolma 3 Mix数据组成: 9T pool → 6T最终训练混合 → 150B实验混合。[OLMo 3, Figure 8] 预训练数据整理流程图。

3.3 Constrained Data Mixing (Olmix)

[OLMo 3, Section 3.4.4, Figure 9] 数据混合优化效果示例: 自然分布 vs 优化后分布 (Figure 9a/9c)，以及对下游评估的改善 (Figure 9b/9d)。
(A) Base Procedure — swarm-based

[OLMo 3, Section 3.4.4] 受RegMix, Data Mixing Laws, CLIMB启发

  1. Swarm: 30M模型, 3B tokens, Dirichlet采样, swarm大小=5×领域数
  2. Regression: 每个代理 → (权重→BPB), 拟合GLM
  3. Optimization: 最小化平均BPB, 重复上限4-7x
(B) Conditional Mixing

[OLMo 3, Section 3.4.4] Chen et al., 2026

  • Round 1: 24个WebOrganizer类别 [Figure 9a]
  • Round 2: 编程语言 (固定web:code=75:25)
  • Round 3: 整合PDF数据 [Figure 9c]

3.4 Quality-Aware Upsampling

[OLMo 3, Figure 10] Quality-aware upsampling曲线 vs flat upsampling曲线。底部40%丢弃，顶部5%重复7次。每个WebOrganizer主题独立生成曲线。

3.6 Midtraining 关键发现

[OLMo 3, Table 7] 领域偏斜混合的trade-off: Math-code-thinking mix提升Math/Code但损害MCQA/GenQA; Gen-QA mix反之。
[OLMo 3, Table 8] Reddit-to-Flashcards microanneal: MCQ +2-3, 但Math和GenQA下降。
[OLMo 3, Table 9] Meta-reasoning/Program-verifiable微退火: Math和Code提升14-20分, 但GenQA/MCQ下降。

补充: MiMo-7B, MiMo-V2-Flash, DeepSeek-V4 NEW

7.1 MiMo-7B

Xiaomi7B dense~25T tokens

数据处理

[MiMo, Section 2.1 "Better Reasoning Data Extraction"] 开发专门HTML提取工具优化数学公式和代码块保留。常用提取器(trafilatura)常丢失这些内容。
[MiMo, Section 2.1 "Multi-Dimensional Data Filtering"] 常用启发式规则过滤器(Penedo et al., 2023, 2024)会错误过滤含数学/代码的高质量网页。改为微调小型LLM作为质量标注器。
[MiMo, Section 2.1 "Synthetic Reasoning Data"] 合成推理数据可训练极多epoch不过拟合(与非推理数据截然不同)。

三阶段混合策略

[MiMo, Section 2.1 "Three-Stage Data Mixture"] 三阶段策略详述。
Stage 1: 所有数据(除合成推理响应)。下采样广告/新闻/招聘。上下文 8,192
Stage 2: 数学+代码增到 ~70%。上下文 8,192 [MiMo, Sec 2.1]
Stage 3: +~10%合成推理响应。上下文 8K→32K [MiMo, Sec 2.3]
[MiMo, Figure 3] MiMo-7B-Base在多个推理benchmark上的Pass@k曲线，显著高于同规模模型。[MiMo, Table 1] 与其他开源7B模型的全面对比。

7.2 MiMo-V2-Flash

Xiaomi309B MoE (15B active)~27T tokens
[MiMo-V2, Section 3 "Pre-Training"] 数据管线沿用MiMo-7B，刻意向长程依赖数据倾斜: 长文本网页、仓库级代码、PR/issue/commit历史。

三阶段Data Scheduler

[MiMo-V2, Section 3.1 "Data Scheduler"] 三阶段详细描述。

Stage 10-22T通用高质量语料0%32K

[MiMo-V2, Section 3.2] 训练超参数: Stage 1 LR warmup→3.2×10-4→cosine至1.0×10-4; Stage 2 cosine至3.0×10-5; MTP loss 0.3→0.1。
[MiMo-V2, Table 5] Base模型全面对比。[MiMo-V2, Table 6] 长上下文性能: NIAH-Multi 256K达96.7%。

7.3 DeepSeek-V4

DeepSeekFlash 284B(13B) / Pro 1.6T(49B)32-33T tokens
[DeepSeek-V4, Section 4.1 "Data Construction"] 在V3基础上构建更多样、更高质量、更长上下文的语料。过滤批量自动生成和模板化内容以降低model collapse风险。数学和编程仍为核心。mid-training加入agentic data。总计32T+ tokens (未公开具体领域比例)。
[DeepSeek-V4, Section 4.2] Flash: 32T, Muon优化器, 序列4K→16K→64K→1M, 前1T dense attention warmup。Pro: 33T, 峰值LR 2.0×10-4。
[DeepSeek-V4, Section 4.2.3] 训练稳定性: Anticipatory Routing和SwiGLU Clamping解决MoE训练中的loss spikes。

Post-Training范式

[DeepSeek-V4, Section 1 Introduction] 两阶段: (1) 独立培养领域专家(数学/编码/agent/指令) — SFT+GRPO; (2) On-policy蒸馏合并 — reverse KL loss。

8.1 原有共识验证 (五篇一致)

[MiMo, Sec 2.1 "Synthetic Reasoning Data"] "synthetic reasoning data can be trained for extremely high number of epochs without overfitting risk" — 这与非推理数据截然不同。

[OLMo 3, Table 10] Thinking traces: Average +1.9

(5) 上下文扩展作为最后阶段

(6) Mid-training阶段普遍采用
所有论文都在主预训练与后训练之间设过渡阶段: 数据分布显著变化 + 学习率衰减 + 为后训练奠基。[daVinci Sec 3.3.2; OLMo 3 Sec 3.5; MiMo Sec 2.1; MiMo-V2 Sec 3.1; DeepSeek-V4 Sec 4.1]

[daVinci, Table 5] Stage 1 constant→cosine→Stage 2 lower constant | [MiMo-V2, Sec 3.2] Stage 1 warmup→constant→cosine, Stage 2 cosine | [DeepSeek-V4, Sec 4.2] warmup→constant→decay
